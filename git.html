gitの使い方

リポジトリgitを含むVCSは基本的な機能として時系列でファイルの記録を管理でき、この記録を管理するデータベースを「リポジトリ」と呼びます。
コミットgitでは、リポジトリに現状のファイルの内容を記録することや、その結果作られた記録のことを「コミット」と呼びます。
ワークツリー「ワークツリー」とは、普通のフォルダのようなものです。普通のフォルダと異なるのは、ワークツリーにあるファイルの変更がGitによって監視されているという点のみ。
ステージングワークツリーがカメラの被写体で、コミットがシャッターを押すことであれば、次にとる写真にどのモチーフを入れるか、構図を決める作業が「ステージング」です。
ブランチ「ブランチ」とは、複数の機能を同時に開発するときに使われる、コミットの系譜の枝話帰れのことを指した概念です。
コンフリクトブランチはファイルの衝突を先伸ばしにすることはできますが、完全に衝突を阻止することは出来ません。同じファイルが2つのブランチでそれぞれ変更され、どうしてもGitでは中身を合体できない、という状況にあると「コンフリクト」が発生します。

github pagesの作り方

　新規リポジトリからの作り方

  1. 空リポジトリを作る
  2. ブランチにindex.htmlを作成
  3. 完成

　既存リポジトリからの作り方

  1. git checkout --orphan gh-pagesでブランチ作成
  2. git rm -rf .ですべてのファイルを削除
  3. pushする
  4. 完成

github pagesの更新の仕方

git add --all全てのファイルをステージング
git commit -m "Update page"変更をコミット
git push -u origin master「master」ブランチをプッシュ
git push -u origin gh-pages「gh-pages」ブランチをプッシュ
